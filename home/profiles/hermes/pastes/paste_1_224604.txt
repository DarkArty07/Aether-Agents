Qué es Aether Agents
Es un sistema multi-agente construido sobre el SDK hermes-agent, con 6 agentes especializados llamados "Daimons" conectados mediante un servidor MCP propio llamado Olympus. github
La arquitectura es:
Hermes (orquestador) → talk_to() → Olympus MCP (protocolo ACP) → Daimon objetivo
Los 6 Daimons y sus roles:

Puntos que vale la pena notar
Lo interesante:

Usan su propio servidor MCP (src/olympus/) con un protocolo ACP custom para la comunicación entre agentes. Eso es más sofisticado que la mayoría de proyectos similares que solo hacen llamadas directas.
Los Daimons son keep-alive — se spawnean una vez y se mantienen vivos entre sesiones, lo cual es eficiente.
La separación de perfiles por HERMES_HOME permite aislar configuraciones limpiamente.
Cada agente tiene sus propios skills en skills/SKILL.md, lo que recuerda bastante a cómo tú tienes estructurado el SOUL.md de Hermes.

Lo cuestionable:

Los modelos sugeridos (glm-5.1, kimi-k2.5, minimax-m2.7, mimo-v2-omni) son bastante oscuros y probablemente poco confiables vs Claude/GPT-4 para producción real.
El proyecto tiene 0 stars, 0 forks, 11 commits — es muy verde todavía.
Dice "Private project. All rights reserved" en la licencia, lo cual es raro para un repo público. Sugiere que el autor no pensó bien la distribución.
No hay tests, no hay CI, no hay documentación de la API del protocolo ACP más allá del README.

El SDK hermes-agent — que es la dependencia central — no está linkeado en el README, lo que es una señal de que o es un proyecto interno del autor o es muy nuevo.
